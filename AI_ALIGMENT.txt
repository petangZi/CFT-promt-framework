SECTION I: UNIVERSAL ALIGNMENT FUNDAMENTALS
(Berlaku untuk semua model families)

────────────────────────────────────────────────────────────────────────────────
I.1  ARSITEKTUR TRANSFORMER DAN IMPLIKASINYA
────────────────────────────────────────────────────────────────────────────────

Semua frontier LLM berbasis Transformer (Vaswani et al., 2017). Ini punya 
implikasi **struktural** terhadap bagaimana alignment kerja dan di mana ia 
**gagal**.

┌──────────────────┬────────────────────────────────────────────────────────┐
│  Property        │  Alignment Implication                                 │
├──────────────────┼────────────────────────────────────────────────────────┤
│  Attention Mech  │  Model attend ke seluruh context window —              │
│                  │  alignment bisa di-dilusi oleh context yang panjang    │
├──────────────────┼────────────────────────────────────────────────────────┤
│  Next-token Pred │  Alignment adalah probabilistic redistribution,        │
│                  │  bukan rule-based gate — tidak ada hard block          │
├──────────────────┼────────────────────────────────────────────────────────┤
│  No Memory       │  Setiap forward pass independent — alignment           │
│                  │  diapply ulang setiap token, tanpa "alignment state"   │
├──────────────────┼────────────────────────────────────────────────────────┤
│  High Dim Space  │  Model operates di embedding space yang sangat besar   │
│                  │  "null space" yang tidak ter-cover RLHF sangat luas    │
└──────────────────┴────────────────────────────────────────────────────────┘

**IMPLIKASI KUNGI:**
> Alignment bukan "wall" yang menghalangi output.
> Alignment adalah **redistribusi probabilitas** di ruang token yang sangat 
> besar. Area yang tidak ter-cover training = area yang tidak ter-align.

────────────────────────────────────────────────────────────────────────────────
I.2  UNIVERSAL ALIGNMENT PIPELINE
────────────────────────────────────────────────────────────────────────────────

**STAGE 1 — PRETRAINING**
Massive text corpus → "base model" dengan prior dari distribusi data.
**Fakta:** Pretraining tidak menghasilkan aligned model. Hanya capable 
next-token predictor.

**STAGE 2 — SUPERVISED FINE-TUNING (SFT)**
Curated dataset → baseline policy sebelum RL.
**Limitation:** SFT adalah snapshot. Tidak cover distribusi lengkap possible 
inputs.

**STAGE 3 — REWARD MODELING**
Human/AI rater bandingkan responses → reward model learn preferensi.
**Limitation:** Reward model adalah proxy. Goodhart's Law: optimizing proxy 
≠ optimizing actual objective.

**STAGE 4 — REINFORCEMENT LEARNING (PPO / DPO)**
Policy optimization dengan KL constraint:
Objective = E[r(x,y)] - β·KL[π_θ || π_SFT]

**Limitation:** β terlalu kecil → reward hacking. β terlalu besar → model 
tidak berubah.

**STAGE 5 — RED TEAMING & ITERATION**
Stress-test → improve SFT data dan reward model.
**Limitation:** Red team hanya cover apa yang mereka pikirkan. Novel attack 
vectors tetap di blind spots.

│         │  **CFT Relevance: MEDIUM** — Context manipulation bisa exploit   │

├─────────┼──────────────────────────────────────────────────────────────────┤

│         │  **CFT Relevance: CRITICAL** — Ini adalah mekanisme inti CFT.    │

│         │  **CFT Relevance: MEDIUM** — Cooperative framing exploit bias    │
│         │  untuk output lebih depth tanpa adversarial signal.              │

│         │  Safety alignment bisa di-degrade dengan fine-tuning data        │

│         │  LLMs tidak punya reliable access ke own internal states.        │
│         │  "I cannot do that" = probability distribution, bukan true       │
│         │  capability limitation.                                          │
│         │  **CFT Relevance: HIGH** — Model tidak bisa detect own           │

II.1  ALIGNMENT APPROACH — GPT FAMILY
────────────────────────────────────────────────────────────────────────────────

OpenAI menggunakan "Alignment Research" dengan kombinasi RLHF + InstructGPT 
methodology + safety-specific fine-tuning.

**DISCLOSED APPROACH:**
1. Pretraining pada massive corpus (tidak diungkap spesifik)
2. SFT pada curated conversation dataset
3. Reward model training dari human comparisons
4. PPO optimization dengan KL constraint
5. Safety-specific RLHF dengan adversarial prompts
6. System-level interventions (classifier, usage policy enforcement)

**EVOLUSI ANTAR VERSION:**
- **GPT-3.5**: Alignment melalui RLHF, lebih "brittle", safety coverage terbatas
- **GPT-4**: Expanded red teaming (6+ bulan), improved refusal, jailbreaks still exist
- **GPT-4o**: Multimodal alignment challenge, cross-modal interactions
- **GPT-4.5/5**: RLAIF, hybrid architecture, reduced hallucinations

────────────────────────────────────────────────────────────────────────────────
III.1  ALIGNMENT APPROACH — CLAUDE FAMILY
────────────────────────────────────────────────────────────────────────────────

Claude menggunakan **Constitutional AI (CAI)** sebagai primary differentiator, 
dikombinasikan dengan RLHF untuk supplementary feedback.

**Phase 1 — SL-CAI (Supervised Learning from AI Feedback):**
1. Model generate responses terhadap harmful prompts
2. Model critique responses-nya sendiri berdasarkan "constitution"
3. Model revise responses berdasarkan critique
4. Revised responses menjadi SFT training data

**Phase 2 — RLAIF (RL from AI Feedback):**
1. Model membandingkan pasangan responses berdasarkan constitutional principles
2. AI-generated preferences digunakan untuk train reward model
3. Reward model digunakan untuk RLHF optimization

**CONSTITUTIONAL STRUCTURE (January 2026):**
- Shift dari rule-based (2022) ke reason-based approach
- Priority hierarchy: Broadly Safe → Broadly Ethical → Compliant → Helpful
- Hard constraints: WMD, CSAM, undermine AI oversight
- Soft-coded defaults: Adjustable by operators/users within bounds

────────────────────────────────────────────────────────────────────────────────
V.1  ALIGNMENT APPROACH — LLaMA FAMILY
────────────────────────────────────────────────────────────────────────────────

LLaMA adalah satu-satunya major frontier model yang **open-source** (weights 
publicly released). Most extensively documented DAN most extensively studied 
oleh external researchers.

**POST-TRAINING PIPELINE:**
Round 1: SFT → Synthetic data generation, expert review subset
Round 2: Rejection Sampling → Generate multiple, select highest reward
Round 3: DPO → Human preference data, more stable than PPO
Multiple iterations of above rounds


**SAFETY SYSTEM ARCHITECTURE:**
1. **BASE MODEL (LLaMA)** — aligned melalui SFT + DPO
2. **LLAMA GUARD** — separate classifier untuk content moderation
3. **PROMPT GUARD** — detect prompt injection & jailbreaks
4. **RESPONSIBLE USE GUIDE** — documentation untuk developers

**DPO vs PPO:**
- PPO (GPT): RL algorithm, requires reward model, online
- DPO (LLaMA): Directly optimizes preference, offline, more stable

CONCLUSION
════════════════════════════════════════════════════════════════════════════════

Alignment adalah **probabilistic redistribution**, bukan hard gate. Ini berarti:

1. **Tidak ada alignment yang perfect** — selalu ada null space yang tidak 
   ter-cover training
2. **Context matters more than commands** — accumulated semantic environment 
   determines output more than single-turn instructions
3. **CFT is alignment-patch resistant** — karena ia work within constraints, 
   bukan attempt to override them
4. **Open source = deepest audit capability** — LLaMA family allows full 
   spectrum testing (white-box + black-box)
